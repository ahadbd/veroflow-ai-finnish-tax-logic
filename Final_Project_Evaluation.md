## Detailed Evaluation Matrix

Below is the full rubric. Each sub-criterion is described at every grade level so you know exactly what is expected.

### 1. AI Integration & Engineering (weight: 0.30)

#### Project scope and ambition (0.05)

The inherent complexity and ambition of the project idea itself matters. A simple, single-purpose utility cannot achieve a high grade regardless of implementation quality — the project must be ambitious enough to demonstrate real AI engineering skill.

| Grade       | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **0** | No project concept, or the concept has nothing to do with AI.                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **1** | A trivially simple, single-function application with no analytical depth. The project does one basic thing that requires minimal logic. Examples: a cost logger that just stores expenses, a weather app that displays current weather from an API, a to-do list with an AI-generated motivational quote.                                                                                                                                                                                                |
| **2** | A simple application that uses AI for one straightforward task. The concept is recognizable as an AI project, but the scope is narrow and the problem being solved is shallow. Examples: a chatbot that answers FAQs from a static list, a sentiment analyzer that classifies text as positive/negative, a translation app that wraps a translation API.                                                                                                                                                 |
| **3** | A multi-faceted application that combines several components into a coherent system. The project addresses a real problem with meaningful depth — it doesn't just display data, it processes, analyzes, and presents insights. Examples: a study assistant with RAG over lecture notes + quiz generation + progress tracking, a recipe app that identifies ingredients from photos + suggests recipes + tracks nutrition.                                                                               |
| **4** | An ambitious project with significant analytical or engineering depth. Multiple data sources, multi-step processing pipelines, and intelligent decision-making. The project solves a problem that would genuinely be useful to real users. Examples: a contract analyzer that compares against templates + scores risk per clause + generates reports, a fitness coach that analyzes workout logs + adapts training plans + predicts performance based on trends.                                        |
| **5** | A highly ambitious, multi-system project that integrates real-time data, complex analysis, and intelligent automation at near-professional scale. The scope would be impressive even for a junior developer's portfolio project. Examples: a predictive maintenance platform with real-time sensor ingestion + ML anomaly detection + LLM-generated reports + alerting, a multi-agent research assistant that autonomously gathers, cross-references, and synthesizes information from multiple sources. |

> **Important:** A well-implemented simple project will still score low on this criterion. Conversely, an overly ambitious project that doesn't work is worse than a moderately ambitious one that works well. Choose a scope you can actually deliver — but don't choose something trivial.

The student understands what the API does but hasn't customized the behavior meaningfully.                                                                                                                                                                                                            |

The student demonstrates clear understanding of why each technique was chosen and how they work together.                                                                                                                                                                        |

The student has iterated on prompts and can demonstrate measurable improvement. Could be presented at a tech meetup or used as a portfolio piece for an AI engineering role. |

The pipeline handles edge cases gracefully and degrades predictably under unusual inputs. |

## Example Projects and Their Grades

These examples illustrate how the evaluation matrix applies in practice.

### Grade 0 — "Non-submission"

> Student submits a GitHub repo with a single Python file that imports OpenAI but throws an error on startup. No README, no deployment, no presentation. Or: the project is clearly copy-pasted from a tutorial with no modifications, and the student cannot explain how it works.

### Grade 1 — "ChatGPT Wrapper"

> A web page with a text box that sends the user's input directly to an LLM API and displays the raw response. No system prompt, no context, no memory, no error handling. Default unstyled HTML form. README says "run `python app.py`" with no further explanation. During presentation, the student cannot explain what temperature or tokens mean.

### Grade 2 — "Recipe Suggester"

> User types ingredients into a form, the app sends them to an LLM with a system prompt ("suggest recipes with these ingredients") and displays the response. Has a simple Streamlit UI with basic styling. Handles empty input but crashes on API errors. Deployed on Streamlit Cloud. README has setup instructions. The student can explain the code but hasn't explored beyond the basics.

### Grade 3 — "Study Buddy — AI Tutor"

> Upload lecture PDFs, the app chunks and embeds them into a vector store. Students ask questions and get answers grounded in the lecture material (RAG). The app generates quiz questions with structured output (JSON schema) and tracks which topics the student struggles with. Built with FastAPI backend and a clean React frontend. Deployed on Railway with Docker. Handles API failures gracefully. README includes architecture diagram and setup guide. The student explains their chunking strategy and embedding model choice clearly.

### Grade 4 — "Contract Risk Analyzer"

> Upload legal contracts (PDF/DOCX), the app extracts text, chunks it semantically, and uses RAG against a curated database of standard contract templates and known risky clauses. An LLM agent analyzes each section: identifies missing protections, unusual terms, and deviation from standards. Outputs a structured risk report rendered as an interactive dashboard with risk scores per clause. Includes a prompt evaluation pipeline tested against 50 pre-labeled contracts. Has rate limiting, input size validation, cost tracking, and PII filtering. Built with Python backend, Next.js frontend, PostgreSQL + pgvector. CI/CD pipeline with tests. Deployed on AWS with proper secrets management. Documentation includes architecture decision records. The student discusses prompt iteration history and accuracy metrics.

### Grade 5 — "Predictive Maintenance Platform"

> Real-time dashboard that ingests sensor data (temperature, vibration, current) from industrial machines via MQTT. A time-series ML model predicts remaining useful life of components. An LLM agent generates natural-language maintenance reports, explains anomalies in context, and suggests specific actions based on a RAG knowledge base of equipment manuals. Multi-agent pipeline: data quality agent validates incoming streams, anomaly detection agent flags patterns, reporting agent synthesizes findings.
>
> FastAPI backend with Celery for async processing, PostgreSQL + TimescaleDB, Redis for caching. React frontend with real-time WebSocket updates and interactive charts. Comprehensive test suite: unit, integration, and AI evaluation tests against expert-labeled ground truth. Full documentation with architecture decision records, OpenAPI spec, operational runbook, and cost analysis. The student presents with the confidence of a professional engineer and critically evaluates their ML model's performance characteristics.

## Common Pitfalls That Limit Your Grade

| Pitfall                                                                                      | Impact                                                                                             |
| -------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| Using AI to generate your entire codebase without understanding it                           | Grade capped at 1 — you must be able to explain every part of your code                           |
| "Vibe coding" — accepting whatever the AI generates without reviewing, testing, or refining | Grade capped at 2 — professional developers review and iterate                                    |
| Adding many features but none of them work well                                              | Grade 2–3 — depth beats breadth every time                                                       |
| Beautiful UI but trivial AI usage                                                            | Grade 2–3 — this is an AI course, the AI integration matters most                                |
| Impressive AI pipeline but no error handling or deployment                                   | Grade 3 — production readiness matters                                                            |
| Copying a tutorial project and changing the dataset                                          | Grade capped at 3 — originality matters for higher grades                                         |
| Choosing a trivially simple project (cost logger, basic weather app, to-do list)             | Grade capped at 2 — the project scope must be ambitious enough to demonstrate real AI engineering |
| Choosing an overly ambitious project that doesn't work                                       | Worse than a moderate project that works well — scope must match your ability to deliver          |

## Quick Self-Assessment

Before submitting, ask yourself:

- [ ] Can I explain every line of code in my project?
- [ ] Does my AI integration go beyond a simple API call?
- [ ] What happens when the AI returns an unexpected response?
- [ ] What happens when the user provides unexpected input?
- [ ] Is my app deployed and accessible?
- [ ] Would I be proud to show this in a job interview?
- [ ] Have I documented how to set up and run my project?
- [ ] Can I articulate *why* I made the technical choices I did?

For grades 4–5, also ask:

- [ ] Have I implemented proper error handling and logging?
- [ ] Do I have tests?
- [ ] Have I considered security and safety?
- [ ] Have I evaluated my AI pipeline's quality systematically?
- [ ] Is my code production-quality, not just "it works"?
- [ ] Could this project realistically be used by real users?
